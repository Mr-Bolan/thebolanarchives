# content/fragments/

Short thoughts, one-liners, rough ideas, incomplete reflections.
Content type: `fragment`. Route: `/fragments`.

Fragments should be fast to publish (brief section 16) and can later grow into entries or
patterns. Use the compact metadata card.

Maturity statuses common here: `fragment`, `sketch`.

Naming: `slug.mdx`. `README.md` is ignored by the loader.
